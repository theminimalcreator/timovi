# Retrospective — Dashboard do Timovi

## ✅ What went well
- Planejamento multi-papéis (PM+UX+Architect) em 3 rounds foi suficiente para alinhar todas as decisões
- Protótipo high-fidelity antes do código economizou iterações de UI
- Decisão de projeto separado (`timovi-dashboard`) desacoplou do template sem conflitos de escrita
- Breakdown em 7 Issues com DAG de 4 rounds — granularidade certa
- Ponytail gate eliminou over-engineering no breakdown e no execute
- API testada com curl real, TypeScript compilou limpo

## ❌ What didn't go well
- Hooks hardcodaram `http://localhost:3001` — quebrou acesso remoto. Deveria usar URL relativa desde o início
- Wizard CLI usa `readline` síncrono — minimalista mas sem file picker visual. Trade-off OK pra v1

## 🔧 Actions
- Sempre testar acesso remoto quando frontend é servido via rede
- URLs relativas como padrão nos hooks — documentar em CONVENTIONS.md
- Validar JSON de feature.json após edições manuais para evitar erros de parse
