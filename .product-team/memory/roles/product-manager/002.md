# Retrospective — Dashboard do Timovi

## What I learned
- Planejamento em 3 rounds multi-papéis foi suficiente para features de escopo médio
- Protótipo high-fidelity validou layout antes de implementar — economia de retrabalho
- PRD com 5 user stories mapeadas pra 7 issues com DAG claro evitou ambiguidade na execução

## What I'll do differently
- Incluir "teste de acesso remoto" como critério de aceitação padrão quando houver frontend
- Documentar a regra de URLs relativas nos hooks como convenção de projeto
